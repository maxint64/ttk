# ttk

> このプロジェクトは、Codex を使った開発を練習するために作っています。現時点では小さな玩具アプリですが、ちょくちょく手を入れながら、実用に近づけるための骨組みに少しずつ肉をつけていく予定です。

担当君（TanTouKun）は、ひとつのアクティビティの中で複数メンバーが複数の役割を回って担当することを管理するためのウェブアプリです。アクティビティごとに役割とメンバーを登録し、担当履歴をもとに次の担当者を日次でローテーションします。

主な機能:

- アクティビティの作成と削除、アクティビティごとの役割、メンバーの管理
- 担当者の自動ローテーション

## 担当が回る規則

担当はアクティビティごと、役割ごとに独立して管理します。同じアクティビティの中でも、司会と記録などの役割はそれぞれ別の履歴を見て次の担当者を決めます。

ローテーションは、担当日より前にある最新の担当を基準にします。メンバーは登録順に並び、現在の担当者の次のメンバーへ進みます。最後のメンバーの次は最初のメンバーに戻ります。担当日に既に担当がある役割は上書きしません。過去の担当がまだない役割は、自動ローテーションでは担当を作成しません。

手動で担当を入れる場合は、同じ日付・同じ役割の担当を新しいメンバーで置き換えます。

スキップは役割とメンバーの組み合わせに対して設定します。ある役割でスキップしても、別の役割の担当には影響しません。スキップ中のメンバーを手動でその役割に担当させることはできません。

スキップには2種類あります。

- 次回: 次にそのメンバーの担当順が来たときだけ飛ばし、自動で解除します。
- 継続: 手動で解除するまで、その役割では担当順が来るたびに飛ばします。

スキップを設定しても、既に作成済みの担当はその場では移動しません。次のローテーションで、そのメンバーの順番が来たときに反映されます。

例:

メンバーの登録順が `Aさん`、`Bさん`、`Cさん` で、司会の担当が次のように回る場合:

| 担当日 | 前回の担当 | スキップ設定 | 作成される担当 |
| --- | --- | --- | --- |
| 2026-05-24 | Aさん | なし | Bさん |
| 2026-05-25 | Bさん | Cさんを次回スキップ | Aさん |
| 2026-05-26 | Aさん | Bさんを継続スキップ | Cさん |
| 2026-05-27 | Cさん | Bさんを継続スキップ | Aさん |

2026-05-25 の `Cさんを次回スキップ` は、その日にCさんの順番が来た時点で自動解除されます。一方、`Bさんを継続スキップ` は手動で解除するまで残るため、2026-05-26 以降もBさんの順番が来るたびに飛ばします。

## Docker で起動

```bash
make docker-up
```

停止する場合:

```bash
make docker-down
```

日次ローテーションは、crontab から 0 時に実行します。

```cron
0 0 * * * cd /path/to/ttk && make docker-rotate
```

日付を指定して手動実行する場合:

```bash
make docker-rotate DATE=2026-05-24
```

スモークテストを実行する。

> Docker のDBをテストデータで初期化し、確認後にDockerサービスを停止します。

```bash
make docker-smoke
```

簡易性能試験を実行する。

> Docker のDBを空に初期化し、確認後にDockerサービスを停止します。デフォルトではアクティビティ1000件、ローテーション100回、担当情報取得1000回を実行します。

```bash
make docker-perf
```

小さい件数で試す場合:

```bash
make docker-perf PERF_ACTIVITIES=20 PERF_ROTATIONS=5 PERF_READS=50 PERF_SKIPS=10
```

## ローカルで起動

> ローカルで起動するために [uv](https://github.com/astral-sh/uv) が必要です。

```bash
make sync
make dev
```

起動後、ブラウザで `http://127.0.0.1:8000` を開きます。

DB migration は Alembic で管理します。アプリ起動やテストデータ投入時にも自動で最新化されますが、手動で適用する場合は次を実行します。

```bash
make migrate
```

日次ローテーションは、crontab から毎日0時に実行します。

```cron
0 0 * * * cd /path/to/ttk && make rotate
```

日付を指定して手動実行する場合:

```bash
make rotate DATE=2026-05-24
```

## テスト

テストコードを実行します。

make test
